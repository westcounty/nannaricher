# 菜根人生 - 游戏平衡性评估设计方案

日期: 2026-03-04

## 目标

通过自动化模拟1000+局游戏，全面评估菜根人生的游戏平衡性，重点分析：
- 33个培养计划的胜率和达成难度
- 8条线路的收益对比
- 经济系统（金钱/GPA/探索值）的平衡性
- 不同玩家人数对平衡性的影响

## 技术方案

基于现有 `test-game-playthrough.mjs` 的 Socket.IO 脚本扩展，复用真实服务端逻辑。

### 架构

```
balance-test/
├── simulator.mjs          # 主入口：批量模拟调度器
├── strategies.mjs         # 4种AI策略定义
├── agent.mjs              # 增强版 PlayerAgent
├── collector.mjs          # 数据收集器
├── analyzer.mjs           # 数据分析器
└── report/                # 输出报告目录
```

### AI策略

| 策略 | 选择逻辑 |
|------|---------|
| random | 所有选择完全随机 |
| greedy_gpa | 优先GPA收益，优先学习线/鼓楼线 |
| greedy_money | 优先金钱收益，优先赚钱线 |
| greedy_explore | 优先探索值，优先探索线/苏州线 |

### 模拟矩阵

- 玩家人数: 2, 3, 4, 5, 6
- 骰子配置: 1骰子(3000金) / 2骰子(2000金)
- 策略: 随机分配 + 固定对局
- 每配置 ~200局，总计 ~1000-1500局

### 数据收集

每局收集: 终局资源、胜负条件、路径统计、资源流水、卡牌统计、事件统计

### 分析维度

1. 培养计划平衡性（达成率、平均达成回合、排名）
2. 线路收益（期望净收益、频率-胜率相关性）
3. 经济系统（汇率验证、通胀曲线、破产率）
4. 基础胜利条件（达成率、占比）
5. 卡牌系统（价值对比、使用率）
6. 人数影响（时长、胜利方式、资源分布）

### 平衡性评判标准

| 指标 | 健康范围 | 警告阈值 |
|------|---------|---------|
| 培养计划胜率 | 10-30% | <5% 或 >50% |
| 基础胜利占比 | 30-60% | <20% 或 >70% |
| 破产率 | 5-15% | >30% |
| 超时结算率 | 10-30% | >50% |

## 输出

- `balance-report.md`: 完整分析报告
- `raw-data.json`: 原始模拟数据
- 总体健康度评分 + 具体调整建议
