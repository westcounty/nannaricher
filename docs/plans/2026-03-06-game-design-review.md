# 菜根人生 — 游戏设计 & UI 全面审查报告

> 审查日期: 2026-03-06
> 审查视角: 资深玩家 + 顶级游戏策划/设计师

## 一、当前亮点

1. **主题契合度极高** — 南京大学的校园元素（浦口、鼓楼、仙林、苏州四校区、各院系培养计划）融入桌游机制，非常有代入感
2. **胜利路径多样性** — 33条培养计划 + 基础胜利条件，战略空间丰富
3. **响应式三端适配** — desktop/tablet/mobile 布局完整
4. **PixiJS + React 架构** — 渲染层与 UI 层分离，技术选型合理
5. **平衡性报告** — 99% 有胜者，71.7% 提前胜利，游戏节奏合理

## 二、核心问题诊断

### A. 界面设计问题（玩家视角）

| # | 问题 | 严重度 | 说明 |
|---|------|--------|------|
| A1 | **信息密度过载** | 高 | 桌面端：Header + Sidebar(玩家卡+培养计划+聊天/日志) + ActionBar + ActionPromptBar + NotificationFeed + 各种Modal，同屏信息太多。玩家不知道该看哪里 |
| A2 | **棋盘视觉层次弱** | 高 | 1900x1900 的 Metro 棋盘上，主环28格+8条支线，站点卡片(80x100)在缩放后非常小。玩家无法快速定位自己在哪，该去哪 |
| A3 | **玩家卡信息不足** | 中 | CompactPlayerCard 只显示姓名+3项数值+badge。看不到位置信息（在哪条线上、离哪个角多远），也看不到培养计划进度。作为对手的威胁感知为零 |
| A4 | **事件弹窗千篇一律** | 中 | 所有事件都用同一个 EventModal 模板：标题+描述+选项按钮。缺少事件类型差异化的视觉设计（大事件 vs 小事件、好事 vs 坏事） |
| A5 | **回合感知弱** | 中 | TurnOverlay 只是一个过场。玩家不知道当前是第几学年第几回合、离毕业还有多远 |
| A6 | **骰子展示4秒过长** | 低 | 每次骰子结果展示4秒，6人游戏每轮24秒纯等骰子动画，严重拖慢节奏 |
| A7 | **卡牌系统隐蔽** | 中 | 手牌在桌面端藏在底部 ActionBar，移动端需要点击才展开。玩家经常忘记自己有手牌可用 |

### B. 机制设计问题（策划视角）

| # | 问题 | 严重度 | 说明 |
|---|------|--------|------|
| B1 | **培养计划极端不平衡** | 高 | 政府管理学院100%胜率 vs 地球科学0%胜率，差距100倍。即使样本不足，分布也严重偏斜（前5名平均60%，后5名平均10%） |
| B2 | **玩家交互不足** | 高 | 90%的游戏体验是"掷骰→走路→触发事件→看弹窗→点确定"的单人PvE循环。投票和链式行动太少，玩家之间缺乏直接博弈 |
| B3 | **GPA+探索基础胜利过于dominant** | 中 | 56.3%的提前胜利来自基础条件(GPA*10+探索>=60)，培养计划胜利只占43.7%。计划系统被边缘化，理论上33种胜利路径，实际上很多人靠堆数值就赢了 |
| B4 | **等待时间过长** | 中 | 6人游戏，轮到你之前要等5个人各掷骰+走路+触发事件+弹窗确认。一轮可能等3-5分钟 |
| B5 | **支线体验被动** | 中 | 进入支线后只能沿线走，没有分叉选择。支线事件也是"看描述→接受结果"。缺少支线内的策略深度 |
| B6 | **经济系统通胀** | 低 | 平均终局金钱3000+，起始就2000-3000。金钱增减幅度相对于基数太小，经济决策的张力不够 |
| B7 | **后期乏味** | 中 | 培养计划在第2-4年才选，但第1年（6回合）的体验与后面几乎相同。缺少成长曲线，游戏从头到尾手感一致 |

## 三、优化方案（按优先级排列）

### 方案 1: 界面信息架构重组（P0）

**目标**：让玩家在任何时刻都清楚 3 件事——我在哪、该做什么、谁在威胁我

具体改动：
- **棋盘**：高亮当前玩家所在格子（大光圈脉冲），在棋盘上直接渲染"下一步可达范围"（掷骰前预览）
- **进度条**：Header 中加入学年进度条（4格，当前高亮），鼠标悬停显示"第2学年 第3回合 / 剩余14回合"
- **侧边栏重构**：玩家卡增加迷你位置指示器（"主环#12" 或 "鼓楼线 3/9"）和培养计划进度条缩略
- **行动提示增强**：当轮到你时，屏幕中央出现醒目的行动提示（不是小条，而是卡片式提示），2秒后自动缩小到底部

### 方案 2: 事件体验差异化（P1）

**目标**：让每个事件"感觉不一样"

具体改动：
- **事件分级**：普通事件（小弹窗/toast）、重要事件（标准Modal）、史诗事件（全屏动画+特效）
- **好/坏事件视觉分化**：好事件绿色光晕+上升动画，坏事件红色闪烁+下坠动画
- **选择类事件增强**：选项卡片化（不是按钮列表），每个选项显示预估影响值（如 "+300 money / -0.2 GPA"），让决策有信息支撑
- **支线入口动画**：进入支线时播放"地铁进站"的短动画（0.5秒），配合线路主题色

### 方案 3: 玩家交互增强（P1）

**目标**：让这真正变成一个"和朋友玩"的游戏，而不是"和朋友一起各自玩"

具体改动：
- **交易系统**：允许玩家之间交易金钱和手牌（每回合限1次）
- **快速反应卡**：部分手牌可以在别人的回合打出（类似UNO的反应机制），如"反弹""拦截"
- **排行榜实时展示**：侧边栏始终显示胜利条件达成进度排名，制造紧迫感
- **表情/快捷语音**：6个预设表情按钮（"不！""哈哈""救命""GG"等），减少打字门槛

### 方案 4: 培养计划平衡调整（P1）

**目标**：让每个计划的实际胜率落在 20%-50% 区间

具体改动：
- **削弱高胜率计划**：政府管理学院、匡亚明学院条件加严
- **增强低胜率计划**：地球科学、环境学院、化学化工等条件放宽或增加被动能力
- **基础胜利条件上调**：GPA*10+探索>=60 -> >=75，逼迫玩家更依赖培养计划
- **动态平衡**：如果某计划连续3局无人达成，下一局该计划获得"热门加成"（能力小幅增强）

### 方案 5: 节奏优化（P2）

**目标**：减少等待，增加每秒有效决策密度

具体改动：
- **骰子展示缩短**：4秒 -> 2秒（自己）/ 1.5秒（他人），或可在设置中跳过
- **非当前玩家小事件改为 toast**：别人的普通事件不弹Modal，只在通知栏闪一下
- **同时行动阶段**：每学年开始时增加一个"同时选择"阶段（如同时选培养计划），减少串行等待
- **自动模式**：断线玩家和AI托管的行动应在1秒内完成，不做动画

### 方案 6: 成长曲线设计（P2）

**目标**：让游戏前中后期体验不同

具体改动：
- **大一(Round 1)**：新手保护期，事件影响减半，重点探索棋盘
- **大二(Round 2)**：选培养计划，解锁主动能力，开始有策略方向
- **大三(Round 3)**：事件收益/风险翻倍，支线经验卡解锁高级效果
- **大四(Round 4)**：终局冲刺——每回合结束检查胜利条件，增加"毕业论文"事件（全员同时挑战）

## 四、Quick Wins（可快速落地的小改进）

1. **骰子动画从4s改为2s** — GameScreen.tsx:111
2. **玩家卡加位置信息** — CompactPlayerCard 显示位置（主环#N 或 线路名 M/N）
3. **Header 加学年进度** — CompactHeader 中渲染 Round N/4，用4个圆点表示
4. **高亮当前玩家棋子** — PlayerLayer 中增加更大更亮的光圈
5. **事件弹窗好坏分色** — EventModal 根据 effects 正负切换绿/红主题

## 五、平衡性数据摘要

- 总模拟: 100局, 99%有胜者
- 提前胜利: 71.7% (其中基础56.3%, 计划43.7%)
- 平均游戏回合: 16.4 (中位数15.5)
- 培养计划胜率分布: 0% ~ 100% (严重不平衡)
- 平均终局综合分: ~44 (目标60, 差距较大说明基础胜利条件可能偏低)
- 探索值与胜率相关性最高(0.364)
