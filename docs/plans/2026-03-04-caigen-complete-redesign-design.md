# 第二部分：棋盘地图设计

## 2.1 地图布局总览

棋盘为经典方形布局，28格围成一圈，4个角落格将棋盘分为4条等长的边（每边7格含1个起始角落格+6个普通格）。8条支线从各边的入口格向棋盘内部延伸，每边2条支线。

**角落索引**: 0（起点）、7（校医院）、14（鼎）、21（候车厅）

**移动方向**: 逆时针（从起点向左移动）

### 设计决策：会议中心

规则书在"四角方格"下列出5个特殊格（起点、校医院、鼎、会议中心、候车厅），但方形棋盘仅有4个角。参考墨刀地图和代码实现，**会议中心作为仙林线的内部事件格**处理（仙林线第1格），不占据主地图角落位。规则书中会议中心的金额标注为"？"（未定），进一步印证其为草稿阶段的设计，在正式版中已归入仙林线。

# 第六部分：核心游戏机制

## 6.1 基础机制

| 机制 | 规则 |
|------|------|
| 资源 | 金钱(💰)、GPA(📚)、探索值(🧭)，数值模型：1探索=0.1GPA=100金钱 |
| 起始配置A | 两个骰子，金钱2000，GPA3.0，探索值0 |
| 起始配置B | 一个骰子，金钱3000，GPA3.0，探索值0 |
| 失败条件 | 金钱<0（等于0可继续），GPA和探索值最小为0 |
| 基础胜利 | GPA×10 + 探索值 ≥ 60 |
| 额外胜利 | 满足任意一项已保留的培养计划条件 |
| 培养计划上限 | 每人至多保留2项 |
| 确认培养计划 | 每6回合可确认1项（确认后不可替换），确认时可移动到任意线起点（需交入场费，经过起点不领取低保） |

| 规则 | 说明 |
|------|------|
| 骰子超出线路终点 | 停在终点执行经验卡（骰子剩余步数作废） |

| 食堂线获得经验卡后 | 获得"宠辱不惊"后，食堂线变为可选进入 |

## 6.5 多人互动系统

### 互动类型

```typescript
type PendingActionType =
  | 'choose_option'        // 单人选择（如蒋公的面子二选一）
  | 'multi_vote'           // 全体投票（如四校联动、泳馆常客）
  | 'chain_action'         // 连锁行动（如八卦秘闻、南行玫瑰）
  | 'simultaneous_choice'  // 同时选择（如初雪留痕）
  | 'dice_roll'            // 投骰判定
  | 'target_selection';    // 选择目标玩家

interface PendingAction {
  id: string;
  type: PendingActionType;
  playerId: string | 'all';
  prompt: string;
  options?: ActionOption[];
  targetPlayerIds?: string[];
  responses?: Map<string, ActionResponse>;
  timeoutMs: number;       // 超时自动选择默认
  defaultChoice?: string;
}
```

### 超时机制

- 单人选择：60秒超时
- 多人投票：120秒超时
- 连锁行动：每人30秒
- 超时后自动选择默认选项（通常为第一个选项）
